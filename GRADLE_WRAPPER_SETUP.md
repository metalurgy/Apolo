# Bitacora Pro - Gradle Wrapper Setup

## Files Added

тЬЕ **Gradle Wrapper Files Created:**

1. **[`gradlew`](gradlew)** - Unix/Linux/macOS shell script
   - Executable script for running Gradle on Unix-like systems
   - Automatically downloads Gradle 8.0.2 if not present
   - Location: Project root

2. **[`gradlew.bat`](gradlew.bat)** - Windows batch script
   - Executable script for running Gradle on Windows
   - Automatically downloads Gradle 8.0.2 if not present
   - Location: Project root

3. **[`gradle/wrapper/gradle-wrapper.properties`](gradle/wrapper/gradle-wrapper.properties)** - Wrapper configuration
   - Specifies Gradle version: 8.0.2
   - Specifies download URL for Gradle distribution
   - Location: `gradle/wrapper/`

## Gradle Wrapper JAR

**Note:** The `gradle/wrapper/gradle-wrapper.jar` file is a binary JAR file that cannot be created via text file tools. This file is typically:
- Generated by running `gradle wrapper` command in an existing Gradle project
- Downloaded automatically by the wrapper scripts on first run
- Committed to version control in real projects

## How to Complete Setup

### Option 1: Download Pre-built Wrapper JAR (Recommended)

If you have an existing Android project with Gradle Wrapper, copy the `gradle/wrapper/gradle-wrapper.jar` file from that project to this project's `gradle/wrapper/` directory.

### Option 2: Generate Wrapper JAR

If you have Gradle installed locally:

```bash
# Navigate to project root
cd /path/to/Bitacora\ Pro

# Generate the wrapper (requires Gradle to be installed)
gradle wrapper --gradle-version 8.0.2
```

This will create the missing `gradle/wrapper/gradle-wrapper.jar` file.

### Option 3: Use Android Studio

1. Open the project in Android Studio
2. Android Studio will automatically detect and set up the Gradle Wrapper
3. The `gradle/wrapper/gradle-wrapper.jar` will be created automatically

## Build Commands

Once the `gradle-wrapper.jar` file is in place, you can build using:

### On Windows (Command Prompt or PowerShell):
```bash
gradlew.bat clean assembleDebug
```

### On macOS/Linux/Git Bash:
```bash
./gradlew clean assembleDebug
```

## Troubleshooting

### Issue: "gradle-wrapper.jar not found"
**Solution:** Download or generate the JAR file as described above.

### Issue: "Permission denied" on macOS/Linux
**Solution:** Make gradlew executable:
```bash
chmod +x gradlew
```

### Issue: "JAVA_HOME is not set"
**Solution:** Set JAVA_HOME environment variable:
```bash
# On Windows
set JAVA_HOME=C:\Program Files\Java\jdk-17

# On macOS/Linux
export JAVA_HOME=/Library/Java/JavaVirtualMachines/jdk-17.jdk/Contents/Home
```
