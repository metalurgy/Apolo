## Part A: PDF Report Generation

### Overview
Added professional PDF report generation for each job using Android's built-in `PdfDocument` API. No third-party PDF libraries required.

### Files Created
- **`app/src/main/java/com/bitacora/pro/reports/JobPdfReportGenerator.kt`** (new)
  - Generates professional PDF reports with job metadata, agenda, and evidence
  - Handles image thumbnails with automatic downsampling
  - Graceful error handling for missing/corrupted images
  - Clean, professional layout with teal accent colors

### Files Modified
- **`app/src/main/java/com/bitacora/pro/data/storage/StorageManager.kt`**
  - Added `getReportsDir(jobId: String): File` - Gets/creates reports directory
  - Added `getReportFile(jobId: String, fileName: String): File` - Gets specific report file
  - Added `getReportFileUri(jobId: String, fileName: String): Uri?` - Gets FileProvider URI for safe sharing

- **`app/src/main/java/com/bitacora/pro/ui/screens/JobDetailScreen.kt`**
  - Added PDF report section with "Generar Reporte PDF" button
  - Added state management for generated reports
  - Added "Abrir Reporte" (Open) action
  - Added "Compartir Reporte" (Share) action
  - Integrated with existing FileProvider for safe file access

### PDF Report Contents

#### Header
- App title: "Bitacora Pro"
- Report title: "Reporte de Trabajo"

#### Job Metadata Section
- Título del trabajo
- Cliente
- Teléfono
- Servicio
- Estado del trabajo
- Fecha de creación
- Última actualización

#### Agenda Section
- Pending items (Pendiente) with orange accent
- Completed items (Completado) with green accent
- Due text (Vencimiento) if present
- Description if present

#### Evidence Section
- Evidence type and category
- File name (if available)
- Creation date
- Text preview for TEXT evidence (first 80 characters)
- Image thumbnail for IMAGE evidence (auto-downsampled if > 200px)
- Graceful fallback if image cannot be decoded

#### Footer
- "Generated by Bitacora Pro v0.5"
- Generation date/time

### PDF Storage
- Location: `filesDir/jobs/<jobId>/reports/`
- File naming: `report_<jobId>_<timestamp>.pdf`
- Safe access via FileProvider (no direct file sharing)

### PDF Actions
1. **Generar Reporte PDF** - Generates new PDF report
2. **Abrir Reporte** - Opens PDF in Android PDF viewer (Intent.ACTION_VIEW)
3. **Compartir Reporte** - Opens Android share sheet (Intent.ACTION_SEND)

### Technical Details
- Uses `android.graphics.pdf.PdfDocument` (built-in API)
- A4 page size (595x842 points)
- Multi-page support (automatic page breaks)
- Image downsampling to prevent memory issues
- Exception handling for image decoding failures
- Proper MIME type handling (application/pdf)
- FLAG_GRANT_READ_URI_PERMISSION for safe sharing

## Known Limitations

1. **Image Thumbnails in PDF:** Large images are automatically downsampled to 200px max dimension to prevent memory issues
2. **PDF Pages:** Multi-page support is automatic but not explicitly tested with very large jobs
3. **Text Preview:** Evidence text is limited to 80 characters in PDF for readability
4. **Image Decoding:** If an image file is corrupted, the PDF generation continues without the thumbnail (graceful fallback)
